## üöÄ Quick Start Guide for Developers

### Step 1: Get a Claude API Key

1. Visit [Anthropic Console](https://console.anthropic.com/)
2. Sign up or log in
3. Navigate to **API Keys** section
4. Click **Create Key**
5. Copy your API key (keep it secure!)
6. Add billing information (required for API usage)

### Step 2: Clone and Install

```bash
# Clone the repository
git clone <repository-url>
cd indianhistorybite

# Navigate to app directory
cd app

# Install dependencies
npm install
```

**Dependencies installed:**
- `express` - Web server framework
- `axios` - HTTP client for Claude API
- `dotenv` - Environment variable management
- `helmet` - Security headers
- `cors` - Cross-origin resource sharing
- `express-rate-limit` - Rate limiting
- `express-validator` - Input validation
- `crypto-js` - Cryptographic utilities
- `chokidar` - File watching
- `colors` - Terminal colors (for CLI tool)

### Step 3: Configure Environment Variables

Create your environment configuration:

```bash
# Copy the example file
cp .env.example .env
```

Edit `.env` with your favorite text editor:

```bash
# Using nano
nano .env

# Using vim
vim .env

# Or open in VS Code
code .env
```

**Required configuration:**

```env
# REQUIRED: Your Claude API key from Anthropic Console
CLAUDE_API_KEY=sk-ant-api03-your-actual-key-here

# RECOMMENDED: Secure your refresh endpoint with a random key
APP_API_KEY=your_secure_random_string_here

# OPTIONAL: Server port (default: 3001)
PORT=3001

# OPTIONAL: Base path for reverse proxy (default: /indianhistorybite)
BASE_PATH=/indianhistorybite

# OPTIONAL: Runtime directory path (default: ../runtime)
RUNTIME_DIR=../runtime

# OPTIONAL: Node environment (development or production)
NODE_ENV=development

# OPTIONAL: Allowed CORS origins (comma-separated)
ALLOWED_ORIGINS=https://app.gyatso.me
```

**Generate a secure APP_API_KEY:**

```bash
# On Linux/macOS
openssl rand -hex 32

# On any platform with Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Step 2: Deploy to Server

#### Option A: Traditional Linux Server (systemd)

**1. Copy files to server:**

```bash
# From your local machine
scp -r indianhistorybite user@yourserver.com:/home/user/

# SSH into server
ssh user@yourserver.com
cd indianhistorybite/app
npm install --production
```

**2. Create systemd service file:**

```bash
sudo nano /etc/systemd/system/indianhistorybite.service
```

**Service file content:**

```ini
[Unit]
Description=Indian History Bite - AI-powered history stories
After=network.target

[Service]
Type=simple
User=yourusername
WorkingDirectory=/home/yourusername/indianhistorybite/app
ExecStart=/usr/bin/node src/server.js
Restart=on-failure
RestartSec=10
StandardOutput=syslog
StandardError=syslog
SyslogIdentifier=indianhistorybite

# Environment
Environment=NODE_ENV=production
EnvironmentFile=/etc/indianhistorybite/.env

[Install]
WantedBy=multi-user.target
```

**3. Set up environment file:**

```bash
# Create secure directory for env file
sudo mkdir -p /etc/indianhistorybite
sudo cp .env /etc/indianhistorybite/.env
sudo chmod 600 /etc/indianhistorybite/.env
sudo chown yourusername:yourusername /etc/indianhistorybite/.env
```

**4. Start the service:**

```bash
# Reload systemd
sudo systemctl daemon-reload

# Enable service to start on boot
sudo systemctl enable indianhistorybite

# Start the service
sudo systemctl start indianhistorybite

# Check status
sudo systemctl status indianhistorybite

# View logs
sudo journalctl -u indianhistorybite -f
```

**5. Configure Nginx reverse proxy:**

```bash
sudo nano /etc/nginx/sites-available/indianhistorybite
```

**Nginx configuration:**

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    # Redirect HTTP to HTTPS (after SSL setup)
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    # SSL certificates (use certbot for Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    location /indianhistorybite {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**6. Enable and reload Nginx:**

```bash
# Enable site
sudo ln -s /etc/nginx/sites-available/indianhistorybite /etc/nginx/sites-enabled/

# Test configuration
sudo nginx -t

# Reload Nginx
sudo systemctl reload nginx
```

**7. Set up SSL with Let's Encrypt:**

```bash
# Install certbot
sudo apt install certbot python3-certbot-nginx

# Get certificate
sudo certbot --nginx -d yourdomain.com

# Auto-renewal is set up automatically
sudo certbot renew --dry-run
```

## üêõ Troubleshooting

### Common Issues and Solutions

#### 1. API Key Errors

**Problem**: `API key not configured` or authentication errors

**Solutions**:
```bash
# Verify .env file exists
ls -la .env

# Check API key is set
grep CLAUDE_API_KEY .env

# Verify API key format (should start with sk-ant-)
# Get new key from https://console.anthropic.com/
```

#### 2. Port Already in Use

**Problem**: `Error: listen EADDRINUSE: address already in use :::3001`

**Solutions**:
```bash
# Find process using port 3001
lsof -i :3001
# or
netstat -tulpn | grep 3001

# Kill the process
kill -9 <PID>

# Or change port in .env
PORT=3002
```

#### 3. Module Not Found

**Problem**: `Error: Cannot find module 'express'`

**Solution**:
```bash
# Reinstall dependencies
cd app
rm -rf node_modules package-lock.json
npm install
```

#### 4. File Watching Not Working

**Problem**: Prompt changes don't trigger new stories

**Solutions**:
```bash
# Check if prompt.txt exists
ls -la ../runtime/data/prompt.txt

# Verify file permissions
chmod 644 ../runtime/data/prompt.txt

# Test manual refresh
curl -X POST http://localhost:3001/indianhistorybite/api/refresh \
  -H "x-api-key: your_key"
```

#### 5. CORS Errors in Browser

**Problem**: `Access-Control-Allow-Origin` errors

**Solution**:
```bash
# Update ALLOWED_ORIGINS in .env
ALLOWED_ORIGINS=https://app.gyatso.me
```

#### 6. Stories Not Displaying

**Problem**: Frontend shows "Loading..." forever

**Solutions**:
```bash
# Check API response
curl http://localhost:3001/indianhistorybite/api/result

# Verify JSON format
curl http://localhost:3001/indianhistorybite/api/result | json_pp

# Check browser console for JavaScript errors
# (F12 ‚Üí Console tab)
```

#### 7. High API Costs

**Problem**: Unexpected billing from Anthropic

**Solutions**:
- Set up rate limiting (already configured)
- Limit refresh endpoint usage
- Use caching for generated content
- Switch to smaller/cheaper model in `server.js`
- Set billing alerts in Anthropic Console

## üìö API Reference

### Public Endpoints

#### GET `/indianhistorybite/`
Main application interface (HTML page)

**Response**: HTML page with story viewer

---

#### GET `/indianhistorybite/api/result`
Get current story data

**Response**:
```json
{
  "response": "{\"name\":\"Rani Lakshmibai\",\"title\":\"The Warrior Queen of Jhansi (1828-1858)\",\"content\":\"Story text...\",\"shareableQuote\":\"Quote here\"}",
  "isProcessing": false,
  "lastModified": "2025-01-15T10:30:00.000Z",
  "error": null
}
```

**Status Codes**:
- `200`: Success
- `500`: Server error

---

#### POST `/indianhistorybite/api/refresh`
Trigger new story generation (protected)

**Headers**:
```
x-api-key: your_app_api_key
```

**Response**:
```json
{
  "message": "Refresh triggered",
  "success": true
}
```

**Status Codes**:
- `200`: Success
- `401`: Missing or invalid API key
- `429`: Rate limit exceeded
- `500`: Server error

**Rate Limits**:
- 10 requests per hour per IP
